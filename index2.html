Responsive Web Design

Введение в адаптивные задачи веб-дизайна

Сегодня существует много типов устройств, которые могут получить доступ к Интернету. Они варьируются от больших настольных компьютеров до небольших мобильных телефонов. Эти устройства имеют разные размеры экрана, разрешения и вычислительную мощность. Адаптивный веб-дизайн - это подход к разработке веб-контента, который отвечает ограничениям различных устройств. Структура страницы и правила CSS должны быть гибкими, чтобы учесть эти различия. В общем, разработайте CSS страницы для своей целевой аудитории. Если вы ожидаете, что большая часть вашего трафика будет поступать от мобильных пользователей, используйте подход «сначала мобильные». Затем добавьте условные правила для экранов большего размера. Если ваши посетители - пользователи настольных компьютеров, разработайте дизайн для больших экранов с условными правилами для меньших размеров. CSS дает вам инструменты для написания различных правил стиля, а затем применять их в зависимости от устройства, отображающего страницу. В этом разделе будут рассмотрены основные способы использования CSS для адаптивного веб-дизайна.

Создать медиа-запрос

Медиа-запросы - это новая техника, представленная в CSS3, которая изменяет представление контента на основе разных размеров области просмотра. Окно просмотра - это видимая область пользователя на веб-странице, которая отличается в зависимости от устройства, используемого для доступа к сайту.

Медиа-запросы состоят из медиа-типа, и если этот медиа-тип соответствует типу устройства, на котором отображается документ, применяются стили. Вы можете иметь столько селекторов и стилей внутри медиа-запроса, сколько хотите.

Вот пример медиазапроса, который возвращает контент, когда ширина устройства меньше или равна 100 пикселей:
@media (max-width: 100px) {/ * Правила CSS * /}
и следующий медиазапрос возвращает контент, когда высота устройства больше или равна 350 пикселей:
@media (min-height: 350px) {/ * Правила CSS * /}
Помните, что CSS внутри медиа-запроса применяется только в том случае, если тип медиа совпадает с типом используемого устройства.

Сделать изображение отзывчивым

Сделать изображения адаптивными с помощью CSS на самом деле очень просто. Вам просто нужно добавить эти свойства к изображению:
img {
  max-width: 100%;
height: auto;
}
Максимальная ширина 100% гарантирует, что изображение никогда не будет шире контейнера, в котором оно находится, а высота auto заставит изображение сохранить исходное соотношение сторон.

Используйте изображение Retina для дисплеев с более высоким разрешением

С увеличением числа подключенных к Интернету устройств их размеры и технические характеристики различаются, а дисплеи, которые они используют, могут различаться как внешне, так и внутренне. Плотность пикселей - это аспект, который может отличаться на одном устройстве от других, и эта плотность известна как Pixel Per Inch (PPI) или Dots Per Inch (DPI). Самый известный такой дисплей - «Retina Display» на последних ноутбуках Apple MacBook Pro и недавно выпущенных компьютерах iMac. Из-за разницы в плотности пикселей между дисплеями «Retina» и «Non-Retina» некоторые изображения, которые не были сделаны с использованием дисплея с высоким разрешением, могут выглядеть «пикселизированными» при визуализации на дисплее с высоким разрешением.

Самый простой способ правильного отображения изображений на дисплеях с высоким разрешением, таких как «Retina Display» в MacBook Pro, состоит в том, чтобы определить их ширину и высоту как половину от исходного файла. Вот пример изображения, которое использует только половину исходной высоты и ширины:
<style>
img { height: 250px; width: 250px; }
</style>
<img src="coolPic500x500" alt="A most excellent picture">

Сделать типографику отзывчивой

Вместо того, чтобы использовать em или px для определения размера текста, вы можете использовать единицы просмотра для адаптивной типографии. Единицы просмотра, такие как проценты, являются относительными единицами, но они основаны на разных элементах. Единицы видового экрана относятся к размерам видового экрана (ширина или высота) устройства, а проценты - к размеру родительского элемента контейнера.

Четыре различных окна просмотра:
vw (viewport width - ширина области просмотра): 10vw будет 10% ширины области просмотра.
vh (viewport height - высота окна просмотра): 3vh будет 3% от высоты окна просмотра.
vmin (viewport minimum - минимум области просмотра): 70vmin будет 70% меньшего размера области просмотра (высота или ширина).
vmax (viewport maximum - максимум окна просмотра): 100vmax будет 100% большего размера окна просмотра (высота или ширина).
Вот пример, который устанавливает тег body в 30% ширины области просмотра.
body { width: 30vw; }

Введение в CSS Flexbox

Пользовательский интерфейс веб-сайта («UI») состоит из двух компонентов. Во-первых, это визуальные элементы, такие как цвета, шрифты и изображения. Во-вторых, есть размещение или позиционирование этих элементов. В адаптивном веб-дизайне макет пользовательского интерфейса должен поддерживать множество различных браузеров и устройств, обращающихся к контенту.

CSS3 представил Flexible Boxes, или flexbox, для создания макетов страниц для динамического интерфейса. Это режим макета, который упорядочивает элементы предсказуемым образом для экранов разных размеров и браузеров. В то время как несколько новые, все популярные современные браузеры поддерживают flexbox. В этом разделе описывается, как использовать flexbox и различные варианты макета, которые он предлагает.

Используем свойство display: flex для позиционирования двух блоков

В этом разделе используются чередующиеся стили вызова, чтобы показать, как использовать CSS для гибкого позиционирования элементов. Сначала задача объяснит теорию, затем практическая задача с использованием простого компонента твита будет применять концепцию flexbox.
Размещение свойства CSS display: flex; [в родительском элементе (контейнере)] позволяет использовать другие свойства flex для создания адаптивной страницы.

Используйте свойство flex-direction, чтобы создать строку

Добавление display: flex к элементу превращает его в flex-контейнер. Это позволяет выровнять любые дочерние элементы этого элемента в строки (rows) или столбцы (columns). Вы делаете это, добавляя свойство flex-direction к родительскому элементу и устанавливая его в строку или столбец. Создание строки будет выравнивать дочерние элементы по горизонтали, а создание столбца выравнивает дочерние по вертикали.

Другие варианты для flex-direction - row-reverse (обратный ряд) и column-reverse (обратный столбец).

Примечание. Значением по умолчанию для свойства flex-direction является row (строка).

Выравнивание элементов с помощью свойства justify-content

Иногда (гибкие) flex элементы внутри (гибкого)flex контейнера не заполняют все пространство в контейнере. Распространено желание рассказать CSS, как выровнять и распределить flex элементы определенным образом. К счастью, свойство justify-content имеет несколько вариантов сделать это. Но сначала, есть некоторая важная терминология, которую нужно понять, прежде чем рассматривать эти варианты.

Напомним, что установка flex-контейнера в виде строки размещает flex-элементы рядом друг с другом слева направо. Flex-контейнер, заданный в виде столбца, размещает flex элементы в вертикальной стопке сверху вниз. Каждое направление, в котором располагаются flex элементы, называется главной осью. Для ряда это горизонтальная линия, которая прорезает каждый элемент. А для столбца главная ось представляет собой вертикальную линию через элементы.

Существует несколько вариантов размещения гибких элементов вдоль линии, которая является главной осью. Одним из наиболее часто используемых является justify-content: center ;, который выравнивает все элементы flex по центру внутри контейнера flex. Другие варианты включают в себя:

flex-start: выравнивает элементы по началу flex-контейнера. Для ряда это выстраивает элементы слева от контейнера. Для столбца это помещает элементы в верх контейнера. Это выравнивание по умолчанию, если не указано justify-content.
flex-end: выравнивает элементы по концу flex-контейнера. Для ряда это выстраивает элементы справа в контейнере. Для столбца это помещает элементы в нижнюю часть контейнера.
space -between: выравнивает элементы по центру главной оси, с дополнительным пространством между элементами. Первый и последний элементы помещаются на самый край гибкого контейнера. Например, в строке первый элемент находится с левой стороны контейнера, последний элемент - с правой стороны контейнера, затем оставшееся пространство равномерно распределяется между другими элементами.
space-around: аналогично space-between, но первый и последний элементы не привязаны к краям контейнера, пространство распределено вокруг всех элементов с полупространством на любом конце flex контейнера.
space-evenly: равномерно распределяет пространство между flex элементами с полным пространством на обоих концах flex контейнера.

Выравнивание элементов с помощью свойства align-items

Свойство align-items похоже на justify-content. Напомним, что свойство justify-content выравнивает элементы flex вдоль главной оси. Для строк главная ось является горизонтальной линией, а для столбцов - вертикальной.

Flex контейнеры также имеют поперечную ось, противоположную главной оси. Для строк поперечная ось является вертикальной, а для столбцов поперечная ось является горизонтальной.

CSS предлагает свойство align-items: для выравнивания flex-элементов вдоль поперечной оси. Для строки это говорит CSS, как перемещать элементы во всей строке вверх или вниз внутри контейнера. И для столбца, как протолкнуть все элементы влево или вправо в контейнере.

Доступны различные значения для align-items:

flex-start: выравнивает элементы по началу flex-контейнера. Для строк это выравнивает элементы по верху контейнера. Для столбцов это выравнивает элементы слева от контейнера.
flex-end: выравнивает элементы по концу flex-контейнера. Для строк это выравнивает элементы в нижней части контейнера. Для столбцов это выравнивает элементы справа от контейнера.
center: выровнять элементы по центру. Для строк это выравнивает элементы по вертикали (равное пространство над и под элементами). Для столбцов это выравнивает их по горизонтали (равное пространство слева и справа от элементов).
stretch: растянуть элементы, чтобы заполнить гибкий контейнер. Например, элементы строк растягиваются, чтобы заполнить гибкий контейнер сверху вниз. Это значение по умолчанию, если не указано значение align-items.
baseline: выровнять элементы по их базовой линии. Базовая линия - это текстовая концепция, думайте о ней как о строке, на которой располагаются буквы.

Используйте свойство flex-wrap для переноса строки или столбца

У CSS flexbox есть возможность разбивать гибкий элемент на несколько строк (или столбцов). По умолчанию гибкий контейнер поместит все гибкие элементы вместе. Например, все строки будут в одной строке.

Однако использование свойства flex-wrap (wrap - заворачивать) говорит CSS обернуть элементы. Это означает, что дополнительные элементы перемещаются в новую строку или столбец. Точка разрыва, где происходит упаковка, зависит от размера элементов и размера контейнера.

У CSS также есть варианты направления переноса:

nowrap: это настройка по умолчанию, которая не переносит элементы.

wrap: оборачивает элементы слева направо, если они находятся в ряду, или сверху вниз, если они находятся в столбце.

wrap-reverse: оборачивает элементы справа налево, если они находятся в ряду, или снизу вверх, если они находятся в столбце.

Используйте свойство flex-shrink для сжатия элементов

Пока что все свойства в задачах применяются к гибкому контейнеру (родительскому элементу гибких элементов). Однако есть несколько полезных свойств для гибких элементов.

Первое - это свойство flex-shrink (shrink - сжиматься). Когда он используется, он позволяет элементу сжиматься, если гибкий контейнер слишком мал. Элементы уменьшаются, когда ширина родительского контейнера меньше, чем общая ширина всех элементов flex внутри него.

Свойство flex-shrink принимает числа в качестве значений. Чем выше число, тем больше оно будет сжиматься по сравнению с другими элементами в контейнере. Например, если один элемент имеет значение flex-shrink 1, а другой - значение flex-shrink 3, то один элемент со значением 3 будет сжат в три раза больше, чем другой.

Используйте свойство flex-grow для расширения элементов

flex-grow - Противоположностью flex-shrink является свойство flex-grow. Напомним, что flex-shrink контролирует размер элементов при сжатии контейнера. Свойство flex-grow управляет размером элементов при раскрытии родительского контейнера.

Используйте свойство flex-basis, чтобы установить начальный размер элемента

Свойство flex-basis определяет начальный размер элемента, прежде чем CSS выполнит корректировки с помощью flex-shrink или flex-grow.

Единицы, используемые свойством flex-basis, совпадают с другими свойствами размера (px, em,% и т. Д.). Значение автоматически определяет размеры элементов в зависимости от содержимого.

Используйте свойство flex Shorthand

Доступен ярлык для одновременной установки нескольких свойств flex. Свойства flex-grow, flex-shrink и flex-basis могут быть установлены вместе с помощью свойства flex.
Например, flex: 1 0 10px; установит для элемента flex-grow: 1 ;, flex-shrink: 0; и flex-base: 10px ;.
Настройки свойств по умолчанию: flex: 0 1 auto ;.

Используйте cвойство order, чтобы Переставить Элементы

Свойство order (порядок) используется, чтобы сообщить CSS порядок появления элементов flex в контейнере flex. По умолчанию элементы будут отображаться в том же порядке, что и исходный HTML. Свойство принимает числа в качестве значений, и могут использоваться отрицательные числа.

Используйте свойство align-self

Последнее свойство для flex-элементов - align-self. Это свойство позволяет настраивать выравнивание каждого элемента отдельно, а не настраивать их все сразу. Это полезно, поскольку другие распространенные методы настройки, использующие свойства CSS float, clear и vertical-align, не работают с элементами flex.

align-self принимает те же значения, что и align-items, и переопределяет любое значение, установленное свойством align-items.
