    Тег
    Тег — это специальный символ разметки, который применяется для вставки различных элементов на веб-страницу таких как: рисунки, таблицы, ссылки и др., и для изменения их вида.
    HTML-документ
    Обычный текстовый файл, который может содержать в себе текст, теги и стили. Изображения и другие объекты хранятся отдельно. Содержимое такого файла обычно называется HTML-код.
    Сайт
    Cайт — это набор отдельных веб-страниц, которые связаны между собой ссылками и единым оформлением.

  • Переопределение стилей - Стили можно переопределить: 1) добавив к элементу новый класс и записав его свойство в таблице стилей после исправляемого стиля; 2) добавив элементу атрибут id и записав новое свойство в таблице стилей; 3) добавив style в элемент; 4) добавив !important к нужному свойству в таблице стилей. Каждый последующий способ перекрывает значение предыдущего.
  • rgba - background-color: rgba(45, 45, 45, 0.1). rgb - красный, зеленый, синий (значения от 0 до 255), а - прозрачность ( значение от 1 до 0. 0 - прозрачный, 1 - полный)
  • box-shadow - Свойство box-shadow принимает значения для offset-x (как далеко подвинуть тень от элемента по горизонтали), offset-y (как далеко подвинуть тень от элемента по вертикали), blur-radius (размытие радиуса), spread-radius (радиус распространения) и color (цвет), в таком порядке. Значения размытия и радиуса распространения являются необязательными. С помощью запятых можно создать несколько теней box-shadow для разделения свойств каждого элемента box-shadow. Вот пример CSS для создания нескольких теней с некоторым размытием, в основном с прозрачными черными цветами: box-shadow: 0 10px 20pg rgba (0,0,0,0.19), 0 6px 6px rgba (0,0,0,0.23);
  • opacity - Свойство opacity в CSS используется для регулировки прозрачности или, наоборот, прозрачности для элемента.
    (opacity:0.7)
    Значение 1 непрозрачно, что совсем не прозрачно. Значение 0,5 - это половина прозрачности. Значение 0 полностью прозрачно. Заданное значение будет применяться ко всему элементу, будь то изображение с некоторой прозрачностью, или цвета переднего плана и фона для блока текста.
  • text-transform - Свойство text-transform в CSS используется для изменения внешнего вида текста. Это удобный способ поменять параметры текста без необходимости изменять HTML-код (управляя только CSS-кодом)
    (text-transform: uppercase;) В следующей таблице показано, как различные значения text-transform изменяют пример текста «Transform me»:
    lowercase - transform me
    uppercase - TRANSFORM ME
    capitalize - Transform Me
    initial - использует значение поумолчанию
    inherit - использует значение родительского элемента
    none - поумолчанию использует оригинальный текст
  • font-size - устанавливает селектору размер шрифта: font-size: 68px;
  • font-weight - устанавливает селектору толщину шрифта: font-weight: 500; (100, 200...900)
  • line-height - для изменения высоты каждой строки в блоке текста. Как следует из названия, оно изменяет количество вертикального пространства, которое получает каждая строка текста.
  • hover - при наведении курсора меняет свойство элемента (может быть привязан к любому селектору), например: a:hover {color: red;}(пробел только перед фигурной скобкой!)
  • Relative Position - изменяет относительную позицию элемента. CSS рассматривает каждый элемент HTML как свой собственный блок, который обычно называют блочной моделью CSS. Элементы уровня блока автоматически начинаются с новой строки (например, заголовки, абзацы и элементы div), в то время как встроенные элементы располагаются внутри окружающего контента (например, изображений или интервалов). Расположение элементов по умолчанию таким способом называется нормальным потоком документа, но CSS предлагает свойство position для его переопределения. Когда позиция элемента установлена ​​относительно, это позволяет вам указать, как CSS должен перемещать его относительно его текущей позиции в обычном потоке страницы. Он соединяется со свойствами смещения CSS слева или справа и сверху или снизу. Они говорят, сколько пикселей, процентов или ems, чтобы переместить элемент от того места, где он обычно расположен. В следующем примере абзац смещается на 10 пикселей вверх (от места, где он первоначально находился в потоке):
    p {
    position: relative;
    bottom: 10px;
    }
    Изменение положения элемента на относительное не удаляет его из обычного потока - другие элементы вокруг него по-прежнему ведут себя так, как если бы этот элемент находился в положении по умолчанию. Примечание. Позиционирование дает вам большую гибкость и власть над визуальным макетом страницы. Полезно помнить, что независимо от положения элементов, базовая разметка HTML должна быть организована и иметь смысл при чтении сверху вниз. Именно так пользователи с нарушениями зрения (которые полагаются на вспомогательные устройства, такие как программы чтения с экрана) получают доступ к вашему контенту.
  • Absolute Position - блокирует элемент на месте относительно его родительского контейнера. В отличие от относительного положения, Absolute Position удаляет элемент из обычного потока документа, поэтому окружающие элементы игнорируют его. Свойства смещения CSS (top или bottom и left или right) используются для регулировки положения.
    Один нюанс с абсолютным позиционированием заключается в том, что элемент будет заблокирован относительно своего ближайшего предка. Если вы забыли добавить правило позиции к родительскому элементу (как правило, это делается с помощью position: relative;), браузер будет продолжать искать цепочку и в конечном итоге по умолчанию использовать тег body.
  • Fixed Position - тип абсолютного позиционирования, который блокирует элемент относительно окна браузера. Подобно абсолютному позиционированию, он используется со свойствами смещения CSS, а также удаляет элемент из обычного потока документа. Другие элементы больше не «понимают», где они расположены, что может потребовать некоторых настроек макета в другом месте.
    Одно ключевое отличие между фиксированной и абсолютной позициями состоит в том, что элемент с фиксированной позицией не будет перемещаться, когда пользователь выполняет прокрутку.
    (например, можно сделать навигационную панель, остающуюся на одном месте, добавив смещение со значением 0px (top: 0px; left: 0px;))
  • float - этот инструмент позиционирования фактически не использует position, но устанавливает свойство float элемента. Плавающие элементы удаляются из обычного потока документа и перемещаются влево или вправо от содержащего их родительского элемента. Он обычно используется со свойством width (лучше использовать %% (от родителя)), чтобы указать, сколько горизонтального пространства требует плавающий элемент.
  • z-index - Когда элементы располагаются так, что перекрываются (то есть, используются position: absolute | relative | fixed | sticky), элемент, следующий позже в разметке HTML, по умолчанию будет отображаться поверх других элементов. Однако свойство z-index может указывать порядок расположения элементов друг над другом. Это должно быть целое число (а не десятичное), и более высокoe значениe свойства z-index элемента перемещаeт его выше в стеке, чем элемент с более низким значением.
  • margin: auto; - центрировать элемент (помимо text-align) можно с помощью задания поля (родителя) margin со значением auto.
    /Этот метод работает и для изображений. Изображения являются встроенными элементами по умолчанию, но их можно изменить на блоки, если для свойства display установлено значение block (img {display: block;})./ - ???
  • color - всё о цвете, градиентах и т.д. в этом уроке и последующих.
  • transform: scale(..) - изменяет масштаб элемента в .. раз.
    Свойство transform имеет множество функций, которые позволяют вам масштабировать, перемещать, вращать, наклонять и т. Д. Ваши элементы. При использовании с псевдоклассами, такими как: hover, которые задают определенное состояние элемента, свойство transform может легко добавить интерактивность вашим элементам. Вот пример масштабирования элементов абзаца в 2,1 раза от их первоначального размера, когда пользователь наводит на них курсор:
    p:hover {
    transform: scale(2.1);
    }
    Примечание. Применение преобразования к элементу div также повлияет на любые дочерние элементы, содержащиеся в div.
  • :: before и :: after - Эти псевдоэлементы используются для добавления чего-либо до или после выбранного элемента.
    Для правильного функционирования псевдоэлементов :: before и :: after они должны иметь определенное свойство содержимого. Это свойство обычно используется для добавления таких вещей, как фотография или текст к выбранному элементу. Когда псевдоэлементы :: before и :: after используются для создания фигур, свойство content все еще требуется, но оно установлено в пустую строку.

Доступность

    Доступность (Applied Accessibility)
  • Атрибут alt в теге img - для слабовидящих. включение атрибута alt при использовании тегов img является обязательным. Однако иногда изображения группируются с уже описывающей их подписью или используются только для оформления, фоновые изображения. В этих случаях альтернативный текст может показаться излишним или ненужным. В этих случаях alt имеет пустое значение alt=""
  • h1...h6 - нужно правильно использовать заголовки по смыслу текста, а не из-за размера шрифта (его можно поменять в CSS). На странице должен быть только 1 заголовок H1 !
  • Основные элементы - HTML5 представил ряд новых элементов, которые дают разработчикам больше возможностей, а также включают в себя функции доступности. Эти теги включают в себя main, header, footer, nav, article и section и другие. По умолчанию браузер отображает эти элементы аналогично скромному div. Тем не менее, их использование при необходимости дает дополнительный смысл в вашей разметке. Одно только имя тега может указывать тип информации, которую он содержит, что добавляет семантическое значение этому контенту. Вспомогательные технологии могут получить доступ к этой информации, чтобы предоставить пользователям улучшенную сводку страниц или варианты навигации.
  • Main - элемент используется, чтобы обернуть основной контент, и на каждой странице должен быть только один элемент. Он предназначен для того, чтобы окружать информацию, связанную с центральной темой вашей страницы. Он не предназначен для включения элементов, повторяющихся на разных страницах, таких как навигационные ссылки или баннеры. Основной тег также имеет встроенную функцию ориентира, которую вспомогательные технологии могут использовать для быстрого перехода к основному контенту. Если вы когда-либо видели ссылку «Перейти к основному контенту» вверху страницы, использование основного тега автоматически дает вспомогательным устройствам такую ​​функциональность.
  • article - это еще один из новых элементов HTML5, который добавляет семантическое значение в вашу разметку. Аргумент article является элементом секционирования и используется для переноса независимого, автономного контента. Этот тег хорошо работает с записями в блогах, сообщениями на форумах или новостными статьями.
  • section - Элемент section также является новым в HTML5 и имеет немного другое семантическое значение, чем article. Статья предназначена для автономного контента, а раздел - для группировки тематически связанного контента. Они могут быть использованы внутри друг друга по мере необходимости. Например, если книга - это статья, то каждая глава - это раздел.
  • div - Если между группами контента нет отношений, используйте div.
  • header - используется для переноса вводной информации или навигационных ссылок для родительского тега и хорошо работает с контентом, который повторяется вверху на нескольких страницах. Примечание. Заголовок предназначен для использования в теге body вашего HTML-документа. Это отличается от элемента head, который содержит заголовок страницы, метаинформацию и т. д.
  • nav - это еще один элемент HTML5 со встроенной функцией ориентира для удобной навигации по экрану. Этот тег предназначен для обтекания основных навигационных ссылок на вашей странице.
  • footer - подобно заголовку и навигации, элемент нижнего колонтитула имеет встроенную функцию ориентира, которая позволяет вспомогательным устройствам быстро перемещаться к нему. Он в основном используется для хранения информации об авторских правах или ссылок на связанные документы, которые обычно располагаются внизу страницы.

  • audio - Улучшение доступности аудиоконтента с помощью аудиоэлемента. Атрибут control включает панель управления звуком. Аудиоэлемент HTML5 дает семантическое значение, когда он оборачивает звук или аудиопоток в вашу разметку. Аудиоконтент также нуждается в текстовой альтернативе, чтобы быть доступным для людей с нарушениями слуха или слуха. Это можно сделать с помощью текста на странице или ссылки на стенограмму.
  • figure & figcaption - figcaption идет внутри figure и описывает содержимое изображения, диаграммы.
  • label for - используется для улучшения доступности Form.
    <form>
    <label for="email">Email:</label>
    <input type="text" id="email" name="email">
    </form>
  • fieldset - доступность переключателей. Каждому выбору присваивается метка label с for, привязывающим к идентификатору соответствующего элемента, как описано выше. Для пояснения назначения кнопок используется тег <legend></legend> (аналогично figcaption и figure)
  • Цвета, контрасты(здесь, а также уроки рядом по курсу)
    Текст в теге ссылки (тот, что стоит между a_href и /a) должен быть информативен ("цвета, контрасты" вместо "здесь")
  • accesskey - HTML предлагает атрибут accesskey для указания сочетания клавиш для активации или фокусировки на элементе. Это может сделать навигацию более эффективной для пользователей только с клавиатуры. HTML5 позволяет использовать этот атрибут на любом элементе, но он особенно полезен, когда используется с интерактивными элементами: ссылки, кнопки и элементы управления формой.
    <button accesskey="b">Important Button</button>
    (На мой взгляд - интересный элемент!)
  • tabindex - Используйте атрибут tabindex для добавления фокуса клавиатуры на элемент. Атрибут HTML tabindex имеет три различные функции, относящиеся к фокусу клавиатуры элемента. Когда он находится на теге, это означает, что элемент может быть сфокусирован на. Значение (целое число, которое является положительным, отрицательным или нулевым) определяет поведение.
    ltddiv tabindex = "0"> Мне нужен фокус клавиатуры! </ div>

    Используйте tabindex, чтобы указать порядок расположения клавиш для нескольких элементов. Атрибут tabindex также указывает точный порядок табуляции элементов. Это достигается, когда значение атрибута установлено в положительное число 1 или выше.

    Установка tabindex = "1" приведет к фокусировке клавиатуры на этом элементе в первую очередь. Затем он циклически проходит последовательность указанных значений tabindex (2, 3 и т. Д.), Прежде чем перейти к элементам по умолчанию и tabindex = "0".

    Важно отметить, что когда порядок вкладок установлен таким образом, он переопределяет порядок по умолчанию (который использует источник HTML). Это может сбить с толку пользователей, которые ожидают начать навигацию сверху страницы. Этот метод может быть необходим в некоторых обстоятельствах, но с точки зрения доступности, будьте осторожны перед его применением.
